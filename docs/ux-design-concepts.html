1 — Current Interface Audit

These problems were identified by reading every template, the full CSS, and the navigation structure. They are ranked by impact on usability.

Critical

Navigation overload — 24 items hidden in 5 collapsible groups

Users must open a group, hunt for the item, and remember which group it's in. The accordion forces one group open at a time, so switching contexts requires two clicks. 4 separate "Analysis" type sections (TA Summary, MTF, Analytics, Performance) look like duplicates.

Critical

13-column signals table with no visual hierarchy

Asset, Market, TF, Signal, Entry, SL, T1, T2, R:R, Confidence, Trend, Confluence, Time — equal visual weight on every column. Users can't scan for what matters. At 1280px most columns clip. The most important decision (BUY/SELL + Confidence) competes with administrative data (Market, Time).

Critical

8 KPI cards dilute attention

8 metrics in two rows means nothing feels important. "AI Active: ON" and "System: OK" are health indicators, not trading metrics — they should live in a status bar, not compete with Win Rate and Signal count. The 2×4 grid also breaks into a 4×2 on medium screens, which disrupts reading flow.

Major

Top navbar is crowded with 7 distinct controls

The 56px top bar holds: menu toggle, breadcrumb, ticker ribbon, confidence range slider, WS status badge, theme toggle, notification bell, and user menu. The confidence slider belongs on the signals page, not globally in the navbar. The ticker ribbon occupies 40% of navbar width.

Major

Page header wastes vertical space on every page

Each page opens with a full-width header block (title, subtitle, action buttons) that takes 60–80px before any content. On a 768px laptop this is 8–10% of visible height just for a title. The breadcrumb in the top nav already provides context, making the page title redundant.

Major

Heatmap + Sentiment + Confluence + Signals + 3 Charts all on one page

The dashboard landing page has 7 content sections. Users must scroll past heatmap and sentiment before reaching signals — the core value. No clear visual hierarchy communicates "this is what you should look at first." Everything competes equally.

Major

Filters scattered inside card headers

Each card header contains inline selects (market, timeframe, signal type, search box). This creates a repetitive pattern: scan header → find filter → adjust → wait for content. Filters should be at the page level, not repeated in each card header, to reduce visual noise.

Minor

Font size 13.5px base is too small for a data-heavy interface

Stripe, Linear, and Vercel all use 14–15px base font. At 13.5px, text in dense tables is hard to scan without leaning in. Combined with the tight line-height on data cells, prolonged use causes eye strain.

Minor

No empty states or loading skeletons — just "Loading..."

Every data section shows plain "Loading signals..." text. Skeleton screens (shimmering placeholder shapes) dramatically reduce perceived load time and maintain layout stability so there's no content reflow.

Minor

No keyboard shortcuts / command palette

Power users (the primary audience) rely on keyboard navigation. No ⌘K command palette, no keyboard shortcuts for common actions (refresh, filter, switch timeframe). Every action requires mouse interaction.

Minor

Color-only signal badges

BUY is green, SELL is red — but no icon or shape differentiates them for colorblind users. WCAG 2.1 AA requires information not to be conveyed by color alone. Adding ▲▼ arrows would fix this in one line of CSS.

Minor

No mobile-optimised layout

The 268px sidebar on a 375px screen leaves only 107px for content. The collapsed 58px sidebar still shows and forces the main content into a very narrow column. There's no bottom navigation bar for mobile, which is the standard pattern.

Concept 1

Command Center

A mission-control layout with a collapsible icon rail on the left, a persistent right panel for signal details, and a command palette (⌘K) for zero-click navigation. The hero area is a live chart with overlaid signal annotations. Optimised for power users who spend 4+ hours a day in the platform.

Command Palette

⌘K opens a global search — jump to any page, asset, or action in 2 keystrokes.

🗂

Dual Navigation Rail

48px icon rail always visible. Second panel slides in with context-relevant sub-items only.

📋

Right Detail Panel

Click any signal row to populate the right panel — no page navigation required.

Live Ticker Ribbon

Moved from navbar to a compact top strip. Market data is visible but not competing with actions.

✓ Strengths

  • Highest information density without clutter
  • Fastest navigation for power users
  • Right panel eliminates drill-down navigation
  • Command palette is best-in-class UX
  • Context nav is self-filtering — only shows relevant items

✗ Trade-offs

  • Learning curve — dual rail not immediately obvious
  • Right panel layout challenging on 13" laptops
  • Complex to implement the context-switching nav
  • May feel overwhelming for occasional users

Concept 2

Focus Flow

No sidebar at all. A clean top navigation bar with 5 primary tabs, and the entire viewport is given to content. Each tab has a focused view optimised for that context. Navigation is horizontal, familiar, and requires zero mental model of a sidebar hierarchy. Inspired by the clarity of Stripe Dashboard.

Inspired by Stripe, Vercel, PlanetScale

🗂

Tab-Based Navigation

5 primary tabs eliminate the sidebar entirely. Zero cognitive overhead — users know exactly where they are.

📐

Full-Width Content

Without a sidebar, the content area gains 268px of space — a ~20% increase in usable width.

📊

Hero Metrics

4 large stat cards at the top — immediate answer to "how am I doing?" before any scrolling.

📱

Mobile Natural

Horizontal tabs translate directly to a bottom tab bar on mobile with zero additional code.

✓ Strengths

  • Zero learning curve — tabs are universally understood
  • More content width available
  • Mobile pattern (bottom nav) is built-in
  • Clean, modern aesthetic like Stripe
  • Fastest for casual/occasional users

✗ Trade-offs

  • Fewer visible destinations (only 5 tabs visible)
  • Deep pages require sub-tabs, adding a second navigation layer
  • Loss of sidebar means less glanceable navigation
  • Admin tools buried inside Tools tab

Concept 4

Clarity Dashboard

Generous whitespace, large type, and a hierarchy-first layout. The sidebar is retained but redesigned to be flat (no collapsible groups) with a clear visual hierarchy. Content is organised in clean card sections with breathing room. The most accessible and immediately professional-feeling of all concepts.

Personalised greeting

Flat sidebar

Concept 5

Adaptive Rail

A hybrid: a minimal icon rail that expands into a full sidebar on hover (no click needed), combined with a context-aware top bar that transforms per page. The rail stays at 60px; hovering any icon immediately expands a flyout panel showing just that section's sub-items. The least chrome, most content.

Hover-expand Rail

60px icon rail → hover any icon → flyout panel shows that section's sub-items. No click required, no layout shift.

Implementation

What changes vs current

✓ Navigation

Replace 5 accordion groups with a flat sidebar using label separators. All 24 items visible without any click. Optional hover-expand for extra width.

✓ KPI Cards

Reduce from 8 to 4 critical metrics (Win Rate, Active Signals, Avg Confidence, Open P&L). Move system status to a passive status strip.

✓ Signals Table

Reduce from 13 columns to 7 by merging Asset+Market into one rich cell, removing Trend/Confluence from table (move to detail panel/tooltip).

✓ Typography

Increase base font to 14px. Increase line-height in data tables from 1.4 to 1.6. Larger heading hierarchy (page titles 20px, card titles 14px bold).

✓ Top Navbar

Simplify to: breadcrumb + search (⌘K) + notification bell + user avatar. Move confidence slider to the Signals page. Move ticker to top ribbon.

✓ Filters

Move all page-level filters to a page-level filter bar (below the top nav, above the content). Remove them from individual card headers.

✓ Dashboard Landing

Prioritise: greeting + 4 KPIs + signals table + market pulse. Move heatmap and confluence widget to a secondary tab/sub-page.

✓ Loading States

Replace all "Loading..." text with shimmer skeleton screens that match the final layout shape.
